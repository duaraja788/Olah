Routes

Routes define a chain-to-chain pair and a venue path. In DiazTrade, routes are authored by an operator and then used to stamp quotes for specific sizes. This UI shows route keys and venue hops in a compact table.

Concept

Route key

Deterministic hash: ROUTE(srcChain,dstChain,srcAsset,dstAsset).

Concept

Venue path

Ordered list of venues where offchain executors may seek liquidity across chains.

routeId src dst pair slippage venues status

Venues

Venues are identifiers representing trading endpoints on different chains (DEXs, RFQ desks, routers). DiazTrade stores a venue tag and enabled flag. This UI lists venues and highlights which ones are active.

venueId chain tag enabled

Quotes

A quote is a stamped record for a route and size with an expiry window. Quotes are meant to be consumed by offchain executors and paired with intent hashes from an intent ledger.

quoteId routeId srcAmount expectedDst validUntil digest

Dispatch

Dispatch is a signal: DISPATCH(quoteId, requester, intentHash, chainId, blockNumber). In a production stack, an offchain system would watch these events and coordinate fills.

Security model

Olah is a static interface. It intentionally avoids wallet signing and onchain execution. The Solidity contracts are designed as ledgers and signal desks: they store hashes and emit events; offchain executors handle actual bridging and swaps.

Principle

No browser custody

All actions in this UI are local mock records; nothing is broadcast without your own tooling.

Principle

Explicit roles

Operator/relayer/council roles are segregated. Pausing/sealing gives controlled shutdown behavior.

Risk

Quote misuse

Quotes are data. Always enforce expiry and slippage offchain; never treat UI output as execution proof.

Risk

Venue drift

Venues can change operational details offchain. Use venue tags to bind offchain config and monitoring.

FAQ

Short answers for the demo build.

FAQ

Does Olah connect to a wallet?

No. This page is a static demo. Use your own scripts/tools if you want to call contracts.

FAQ

Why are hashes random?

They represent offchain payloads and route definitions. The UI uses random hex for demo visuals.

FAQ

What’s the difference between quote and dispatch?

Quote = a stamped offer for a route and size. Dispatch = pairing a quote with an intent hash to execute.

FAQ

Where is execution verified?

In your executor stack (offchain). If you use a relayer-attestation ledger, fills can be attested onchain.

Signals

Event-driven execution

A stamped quote is data; a dispatch is a signal. Offchain executors and relayers coordinate fills and (optionally) attest outcomes to an intent ledger. This UI is intentionally read-only and local.

Hint: Use the activity feed as an event timeline during demos.

Signals

What to verify

Always verify: expiry windows, venue enablement, route retirement status, and slippage constraints. Treat hashes as identifiers, not proof of execution.

Hint: If a venue is disabled, retire the route or update the venue path.
